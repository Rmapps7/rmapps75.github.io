§ 3

Nature of the Service — Directory Model

RadioDir operates as a radio station directory and hyperlink service. RadioDir does not host, store, rebroadcast, or retransmit any audio content.

The Station Data provided by this API consists exclusively of:

  • Station names and metadata
  • Publicly accessible stream URLs published by broadcasters themselves
  • Station logos and images as provided by broadcasters
  • Genre, language, and geographic classification data

All stream URLs in the RadioDir directory point directly to the broadcasters' own servers. RadioDir and applications using this API act solely as directory and hyperlink services, redirecting users to content hosted and controlled by the broadcasters themselves.

§ 4

Broadcaster Rights & Takedown Policy

RadioDir respects the rights of all broadcasters. Any broadcaster whose station is listed in the directory may:

  • Request removal of their station from the directory at any time
  • Request correction or update of their station information
  • Register and claim their station through the Broadcaster Portal
  • Opt their station into or out of specific categories

Takedown requests are processed within 24 hours of receipt. Broadcasters may submit requests via the Broadcaster Portal or by emailing takedown@radiodir-api.com.

Upon a valid takedown request, the affected station will be removed from all API responses and any associated station data will be deleted from the RadioDir database.
